# Agent Docs Index

Use this folder as the progressive-disclosure layer for agent guidance.

## Read first

- `project-context.md` — what this project is and what the codebase looks like.
- `workflow.md` — how to work safely and consistently in this repo.
- `roles-and-responsibilities.md` — how to apply the Conductor / Planning / Implementation / Review patterns.
- `testing-and-validation.md` — validation commands, expectations, and test strategy.
- `learning-log.md` — how to record learnings, fixes, and documentation updates after changes.

## Progressive disclosure guidance

1. Start with `project-context.md` for orientation.
2. Open `workflow.md` only when you need to make or review changes.
3. Open `roles-and-responsibilities.md` when deciding which agent behavior to follow.
4. Open `testing-and-validation.md` when validating behavior or diagnosing failures.
5. Open `learning-log.md` when a fix, regression, or documentation improvement should be captured for future work.
